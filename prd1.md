# Product Requirements Document 2
## Eximp & Cloves Infrastructure Limited — Professional Dashboard Redesign & Analytics
### Version 1.0

## 1. Overview

This PRD covers the complete redesign of the admin dashboard and the addition of a full analytics layer. The current dashboard (4 static stat cards + recent invoices table) is replaced with a professional command centre that gives management real-time visibility into company performance across revenue, clients, estates, and sales activity.

This is a **frontend-heavy** PRD. The backend changes are minimal — primarily new query endpoints that aggregate existing data. All charts use **Chart.js** (loaded from CDN). The frontend remains plain HTML/CSS/JS served from FastAPI.

---

## 2. Role-Based Dashboard Behaviour

Two entirely different dashboard experiences based on role:

### Admin Dashboard
Full visibility:
- All revenue KPIs and financial figures
- Sales rep leaderboard
- All charts (revenue trend, estate breakdown, payment status, referral sources)
- Recent activity feed
- Timeframe selector controlling all data

### Staff Dashboard
Operational focus only — no financial figures:
- Pending Verifications count and quick-access button
- Today's new form submissions count
- Clients added this week
- Recent invoices (status only — no revenue totals)
- Quick action buttons (New Invoice, Record Payment, Add Client)
- A motivational summary: "X verifications awaiting your review"

The role check happens in JavaScript on page load using the `/auth/me` endpoint. Staff users never receive financial aggregate data from the API — the backend enforces this too.

## 3. Timeframe Selector

A single control at the top of the admin dashboard that controls **all** KPI cards and charts simultaneously. It does not affect the tables (recent invoices, activity feed) which always show the latest records regardless of timeframe.

### Options
- **This Week** — Monday to today
- **This Month** — 1st of current month to today
- **This Quarter** — Q1 (Jan–Mar), Q2 (Apr–Jun), Q3 (Jul–Sep), Q4 (Oct–Dec)
- **This Year** — January 1st to today
- **Custom Range** — two date pickers (start date, end date), "Apply" button

### Implementation
- Timeframe selector is a styled pill/tab group, not a dropdown
- Selected timeframe is stored in `localStorage` so it persists across page refreshes
- When timeframe changes, all chart data and KPI values re-fetch from the API with the new date range parameters
- All API calls pass `?start=YYYY-MM-DD&end=YYYY-MM-DD` query parameters
- A subtle loading state (spinner overlay on each card/chart) shows while data refreshes

## 5. New Backend Endpoints for Analytics

All endpoints require JWT authentication. All accept `?start=YYYY-MM-DD&end=YYYY-MM-DD` query params. Staff role receives a `403` on financial endpoints.

### 5.1 KPI Summary Endpoint
```
GET /api/analytics/kpis?start=&end=
```
**Response:**
```json
{
  "total_revenue": 12500000,
  "amount_collected": 8750000,
  "outstanding_balance": 3750000,
  "new_clients": 14,
  "plots_sold": 14,
  "avg_deal_size": 892857,
  "collection_rate": 70.0,

  "delta": {
    "total_revenue": 12.5,
```

## 6. Activity Log Table

A new `activity_log` table that records every significant system event. This powers the activity feed on the dashboard.

```sql
CREATE TABLE IF NOT EXISTS activity_log (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    event_type VARCHAR(100) NOT NULL,
    description TEXT NOT NULL,
    client_id UUID REFERENCES clients(id),
    invoice_id UUID REFERENCES invoices(id),
    performed_by UUID REFERENCES admins(id),
    metadata JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_activity_log_created_at ON activity_log(created_at DESC);
ALTER TABLE activity_log ENABLE ROW LEVEL SECURITY;
```

**Where to insert activity log entries (in existing code):**
- Invoice created → log "invoice_created"
- Payment confirmed → log "payment_confirmed"
- Form submission received → log "form_submission"
- Receipt sent → log "receipt_sent"
- Submission rejected → log "submission_rejected"
- Receipt voided → log "receipt_voided"
- New client created → log "client_created"

## 13. Timeframe Selector Implementation

```javascript
// State
let currentTimeframe = localStorage.getItem('ec_timeframe') || 'month';
let customStart = null, customEnd = null;

function getDateRange(timeframe) {
  const now = new Date();
  const pad = n => String(n).padStart(2, '0');
  const fmt = d => `${d.getFullYear()}-${pad(d.getMonth()+1)}-${pad(d.getDate())}`;
  const today = fmt(now);

  if (timeframe === 'week') {
    const day = now.getDay() || 7;
    const monday = new Date(now);
    monday.setDate(now.getDate() - day + 1);
    return { start: fmt(monday), end: today };
  }
  if (timeframe === 'month') {
    return { start: `${now.getFullYear()}-${pad(now.getMonth()+1)}-01`, end: today };
  }
  if (timeframe === 'quarter') {
    const q = Math.floor(now.getMonth() / 3);
    const startMonth = q * 3;
    return { start: `${now.getFullYear()}-${pad(startMonth+1)}-01`, end: today };
  }
  if (timeframe === 'year') {
    return { start: `${now.getFullYear()}-01-01`, end: today };
  }
  if (timeframe === 'custom') {
    return { start: customStart, end: customEnd };
  }
}

async function refreshDashboard() {
  const { start, end } = getDateRange(currentTimeframe);
  localStorage.setItem('ec_timeframe', currentTimeframe);
  // Show loading state on all cards
  setCardsLoading(true);
  // Fetch all data in parallel
  const [kpis, trend, estates, payStatus, referrals, leaderboard] = await Promise.all([
    apiFetch(`/api/analytics/kpis?start=${start}&end=${end}`).then(r => r.json()),
    apiFetch(`/api/analytics/revenue-trend?start=${start}&end=${end}`).then(r => r.json()),
    apiFetch(`/api/analytics/estates?start=${start}&end=${end}`).then(r => r.json()),
    apiFetch(`/api/analytics/payment-status?start=${start}&end=${end}`).then(r => r.json()),
    apiFetch(`/api/analytics/referral-sources?start=${start}&end=${end}`).then(r => r.json()),
    apiFetch(`/api/analytics/rep-leaderboard?start=${start}&end=${end}&limit=10`).then(r => r.json()),
  ]);
  setCardsLoading(false);
  renderKPIs(kpis);
  drawRevenueChart(trend);
  drawEstateChart(estates);
  drawPaymentStatusChart(payStatus);
  drawReferralChart(referrals);
  renderLeaderboard(leaderboard);
}
```

## 14. Delta Calculation (Backend)

For each KPI, the backend must calculate the delta by comparing the current period against the equivalent previous period:

```python
def get_previous_period(start: date, end: date):
    """
    Given a date range, return the equivalent previous period of the same length.
    E.g. March 1–31 → February 1–28
    """
    delta = end - start
    prev_end = start - timedelta(days=1)
    prev_start = prev_end - delta
    return prev_start, prev_end
```

Delta is expressed as a percentage: `((current - previous) / previous) * 100`. If previous is 0, delta is `None` (show "—" in the UI).
