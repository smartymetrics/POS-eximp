# HRM Portal — Full Training Guide

Purpose: A complete, practical training manual for HR users covering portal navigation, user management, leave requests, KPI library and goal setup, and how automated KPI grading works.

Audience: HR administrators, managers, and trainers.

How to use: Use this as a script for a live demo or as a handout. Replace the screenshot placeholders with actual images from your environment.

---

## 1. Quick Start (5 minutes)
- Login to the HRM portal.
-- Overview of main areas: Dashboard, Staff (Users), Leave, Goals, KPI Library, Reports.
-- Goal of this training: be able to create KPI templates, assign goals to staff/departments, and verify automated grading.

## 2. Navigation and Core Pages
- Dashboard: summary cards and quick links.
- Staff / Users: view and edit staff accounts; add staff via People & Org → Employees.
- Leave Requests: approve/reject, view history.
 - Goals: list of active goals, progress bars, Sync Performance button (🔄 Sync Performance).
 - KPI Library: create/edit measurement templates (manual or automated).

## 3. User & Role Basics
- Roles: `hr_admin`, `manager`, `staff` (your environment may include more roles).
- Add a staff user: People & Org → Employees → click `+ Add Staff` → fill name, email, department, set a password, choose roles → click `Create Staff Account`.
- Edit profile: set department, manager, phone number.

## 4. Leave Requests (brief)
- Submit: staff uses the Leave form.
- Approve/Reject: HR views pending, selects action, writes comment.

## 5. KPI Library (Templates)
- Purpose: central definitions of KPIs and whether they're auto‑measured or manual.
- To create:
  - Open Goals & OKRs → Manage KPI Library → + Define New KPI
  - Enter: `KPI Name`, `Target Department`, `Measurement Source` (see list), `Default Unit`, `Description` / Guidance
  - Mark `Active` and Save
- Measurement source examples (auto):
  - `sales_revenue`, `sales_deals_closed`, `sales_collection_rate`, `mkt_leads_added`, `mkt_lead_conversion`, `ops_appointments`, `admin_ticket_esc`
- Manual: choose `manual` if manager will input actuals.

## 6. Creating and Assigning Goals
- Goals page → click `+ Set KPI Goal`, then use `Create & Activate Goal` in the modal
- Assign to an Individual OR Department (either one)
- Choose a KPI template from the library (or type a custom KPI name)
- Set Monthly Target, Unit, Period (month), and Publication Status (Published to show staff)
- Save — the new goal appears in Goals list.

> Note: If you select a template with `measurement_source != manual`, the system will attempt to auto-calculate the actual value.

## 7. Manual vs Automated KPIs (what HR & managers must know)
- Manual KPIs: managers must update actuals (Edit goal → enter actual value).
- Automated KPIs: system computes actuals from live business data (invoices, clients, appointments, tickets).
- The KPI template selector displays the exact UI label: `🤖 Automated Metric` when `measurement_source != "manual"`, or `✍️ Manual Metric` when `measurement_source == "manual"`.

## 8. How KPI Auto‑Grading Works (confirmation & conditions)
- The UI supports automated KPIs, but automation depends on backend services being enabled and configured.
- Requirements for auto‑grading to work correctly:
  1. Database migrations applied (adds `admins.sales_rep_id`, `support_tickets.resolved_at`, and ensures `marketing_contacts.created_by` is recorded). These are backend DB changes.
  2. A backend sync engine (`sync_goal_actuals`) that reads live data, computes KPI values, and writes `actual_value`, `achievement_pct`, `achievement_status`, and `last_synced_at` to `staff_goals`.
  3. A scheduler that runs the sync periodically (or developer provides an API endpoint to run it on demand).
- If all above are in place, automated KPIs will be graded automatically.
- If not, templates and goals still exist, but their `achievement_pct` will remain 0 or blank until the sync is implemented and run.

## 9. Sync Interval & Manual Trigger
- Default interval: the recommended setup runs a nightly sync (once per night). The schedule is defined in the backend scheduler (e.g., `scheduler.py`) and can be adjusted by developers/ops.
 - Manual trigger: HR can click the **🔄 Sync Performance** button in the Goals view to call the sync endpoint (`POST /api/hr/goals/sync`) which requests an immediate refresh. This is useful for testing.
- For near‑real‑time needs, developers can change schedule frequency or add ad-hoc triggers; by default, expect nightly sync.

## 10. Verifying Results (step‑by‑step)
1. Create or pick an automated KPI template (e.g., `sales_revenue`).
2. Create a goal for a staff member for the current month with a reasonable target.
3. Ensure the staff is linked to their `sales_rep` record (see section 11).
4. Click the **🔄 Sync Performance** button; wait ~2–10 seconds and refresh the Goals list.
5. Confirm `achievement_pct` updated, `achievement_status` shows `achieved`/`on_track`/`at_risk`/`behind`, and `last_synced_at` is recent.

## 11. Sales Rep Mapping (important for revenue KPIs)
- Sales KPIs resolve a staff→sales_rep relationship using `admins.sales_rep_id`. If the link is missing, revenue KPIs evaluate to 0.
- Action for HR/ops: confirm with Sales Ops or your development team that each HR staff account is mapped to the correct `sales_rep_id` record in the backend system. This mapping is not configured in the HR portal UI itself.

## 12. Troubleshooting (common issues)
- KPI values remain 0:
  - Check that the KPI template is automated, not manual.
  - Confirm developer migrations and sync engine are deployed.
  - For sales KPIs, confirm the sales rep link exists (`sales_rep_id`).
  - Check period/month alignment — data must fall within the goal’s period.
- Old marketing contacts not counted: contacts created before the marketing fix may have `created_by = NULL` and won't be assigned to staff.
- Tickets or appointments miscounted: ensure `resolved_at` or appointment `scheduled_at` fields exist and are used.

## 13. What HR can configure vs what Developers must do
- HR actions (no code): create templates, create goals, assign staff/departments, publish goals, use the `🔄 Sync Performance` button when needed, confirm sales_rep mapping exists with Sales Ops / Developers.
- Developer actions (required for automation): run DB migrations, implement/enable sync engine (`sync_goal_actuals`), ensure scheduler runs nightly, patch marketing and support handlers to write required columns.

## 14. Admin Demo Script (30 min)
- 0–5m: Portal tour: Dashboard, Staff, Goals, KPI Library.
- 5–12m: Create 2 KPI templates (one manual, one automated) and explain measurement sources.
- 12–18m: Create and assign 3 goals (individual + department), set targets for this month.
- 18–22m: Demonstrate editing a manual KPI actual value and saving.
- 22–28m: Demonstrate automated KPI flow: ensure rep link exists, click the `🔄 Sync Performance` button, verify results.
- 28–30m: Q&A and next steps (linking reps, reporting issues to devs).

## Detailed Sidebar Walkthrough — HR Admin (expanded)
Below are step‑by‑step procedures for the most common HR tasks grouped by sidebar section. Use these during hands‑on training — demonstrate each step in the UI as you read it.

### Overview
- HR Overview (Dashboard)
  1. What it shows: high‑level KPIs (headcount, average score), quick links to Goals, Staff, and Reports.
  2. Common actions: click any KPI card to open the underlying report; use date filter at top to change period.
  3. Trainer tip: start the demo here to orient trainees to current state before drilling into features.
- Task Manager
  1. To create a task: click `Create Task` → enter title, assignee, due date → `Save`.
  2. To manage: open task → use `Assign`, leave internal note, change status to `Mark Complete`.

### Recruitment (common flows)
- Jobs (post a job)
  1. Click `Jobs` → `New Job`.
  2. Fill Title, Department, Hiring Manager, Location, Description, Attachments.
  3. Click `Publish` to make the role live; use `Close Job` when hiring complete.
  4. Tip: use `View Applicants` to screen candidates; click candidate to open application.
- Applications → Shortlist / schedule interview
  1. In `Applications`, filter by job, keyword or stage.
  2. Select candidate → click `Shortlist` or `Reject`.
  3. To schedule interview: `Schedule Interview` → pick interviewer & slot → `Send Invite`.

### People & Org (staff management)
- Employees (add or edit staff)
  1. Click `Employees` → `+ Add Staff` → enter name, email, department, role → `Create Staff Account`.
  2. To edit: open staff profile → `Edit Profile` to change job title, manager, phone, department → `Save`.
  3. To deactivate: `Edit Profile` → `Archive & Deactivate Staff` (use only for leavers).
- Departments & Org Chart
  1. Add department: `Departments` → `Add Department` → name, head, description → `Save`.
  2. Use `Org Chart` to verify reporting lines and spot missing managers.

### Time & Attendance
- Attendance adjustments
  1. Open `Attendance` → search employee/date → click entry → `Adjust Entry` to correct clock times → `Save`.
  2. Approve missing punches: select entries → `Approve Missing Punches`.
- Shift Scheduling
  1. `Shift Scheduling` → `Create Shift` → set owner, start/end times, repeat rules → `Publish Schedule`.
  2. `Swap Shift` allows ad‑hoc swaps between staff (requires approval flow).

### Leave (end‑to‑end)
- Approving leave requests
  1. Open `Leave Requests` → filter `Pending` → click request to view details and attachments.
  2. Click `Approve` or `Reject`; add an optional comment for the employee.
  3. Use `Bulk Approve` to approve multiple pending requests at once (use cautiously).
- Managing balances
  1. `Leave Balances` → search employee → `Adjust Balance` to correct accrual errors → `Save`.
  2. Policy changes: `Leave Policies` → `Create Policy` → define accrual rules and apply to employee groups.

### Performance (detailed — critical for KPI training)
- Goals & OKRs — create and publish a goal
  1. Open `Goals & OKRs` → click `+ Set KPI Goal`, then in the goal dialog click `Create & Activate Goal`.
  2. Assign to `Staff Member` or `Department`. If assigning to staff, select the person; the department auto‑fills.
  3. Choose a KPI Template from the KPI Library (recommended for automated metrics) or enter a custom KPI name.
  4. Set `Monthly Target`, `Unit` (e.g., NGN, deals, %), `Performance Period` (select month), and set `Publication Status` to `Published`.
  5. Click `Create & Activate Goal` — the goal appears in the list with a progress bar.
- Triggering a manual sync and interpreting results
  1. To refresh automated KPIs immediately, click the `🔄 Sync Performance` button in the Goals view.
  2. Wait a few seconds and refresh the list; look at `achievement_pct`, `achievement_status` (Achieved / On Track / At Risk / Behind) and `last_synced_at`.
  3. If `achievement_pct` is 0 for sales KPIs, check `Sales Rep Mapping` (section 11).
- Updating manual KPIs
  1. For `manual` measurement_source, open the goal → `Update Progress` → enter the actual value → `Save`.
- Manager review and signoff
  1. Managers can `Edit` goal details and `Mark Complete` when met. HR can run `Reports` to export team performance.
- 360° Peer Reviews (anonymous option)
  1. Open `Performance` → `360° Peer Reviews` → `+ Launch Review`.
  2. Choose the reviewee, assign peers/direct reports/manager, and set the review deadline.
  3. Check `Anonymous responses` to keep reviewer identities hidden in the final results.
  4. Click `Launch Review`; reviewers receive email invites and their responses remain anonymous if selected.
